What is OSI Model?

OSI or (Open System Interconnection) is a standard ISO model for distributed applications and networking protocols. The control is passed from one layer to the next, starting from the application layer and proceeding to the bottom physical layer.

You can also add the following points:

  • It was developed by ISO International Standard Organization in 1984.
  • It provides a logical framework for data communication.
  • The standards were created to allow different networks to work together efficiently.

In Short

An OSI is a standard model that allows any two systems to communicate even if their architectures are different.

1. Layer-Seven Application

The application layer is the topmost layer in the OSI model of network communication. It provides communication services directly to end-users or applications and enable interaction between software applications and the underlying network, ensuring effective communication between different systems.

For Example:

  • HTTP (HyperText Transfer Protocol)
  • FTP (File Transfer Protocol)
  • SMTP (Simple Mail Transfer Protocol)
  • DNS (Domain Name System) etc.

2. Layer-Sixth Presentation

The Presentation layer in OSI Model, focuses on the formate and the syntax of data sxchanged between applications. It is responsible for translating, encrypting and compressing of data to ensure that devices with different data representations can communicate effectively.

For Example: The Functionalities of Presentation Layer are:

  • Data Encryption
  • Data Compression
  • Translation
  • Formate Conversion etc.

3. Layer-Fifth Session

The Session layer in OSI Model, manages and controls communication sessions between applications. It establishes, maintains, and terminates connections ensuring orderly and synchronized data exchange between systems.

For Example: The Functionalities performed by Session layer are:

  • Dialogue Control
  • Token Management
  • Synchronization
  • Data Recovery

4. Layer-Fourth Transport

The Transport layer in OSI Model, is responsible for the reliable delivery of data between devices on a network. It manages issues like data integrity, error detection and correction and flowcontrol to ensure accurate and efficiently communication.

For Example:

  • TCP (Transmission Control Protocol)
  • UDP (User Datagram Protocol)

5. Layer-Third Network

The Network layer in OSI Model, handles the routing of data between network devices in different networks. It is also responsible for logical addressing, packet forwarding and determining the best path for data to travel from the source to the destination across Interconnected networks.

For Example:

  • IP addresses (Internet Protocol)

6. Layer-Second Data-Link

The data-link layer in OSI Model provides point-to-point or point-to-multipoint communication over a physical channel. Data-Link layer ensures the error free and efficiently transmission of data frames between devices on the same network.

For Example:

  • Ethernet Protocols

7. Layer-First The Physical layer

The Physical layer in OSI Model, is responsible for the physical connection between devices and how bits are transmitted over the network medium,whether through coaxial, copper, or fiber optic cables or through wireless signals. It mainly focuses on the details of voltage levels, frequences, and data rate.

For Example: It includes

  • Twisted-pair cables
  • Fiber Optic cables
  • Coaxial cables
  • Wifi (Wireless Fidelity)
  • Bluetooth
  • Satellites etc...
